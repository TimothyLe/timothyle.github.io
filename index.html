September, 2016

A fully-electric vehicle equipped with brake pedal sensors, throttle calibration, regen calibration, temperature sensors, and more. This begs the question, where can we store and read this information, especially on hardware like the VCU(Vehicle Control Unit, we use the HY-TTC 50)? The solution is EEPROM, a electrical programmable and erasable memory database. The quality of non-volatile memory means that the data stays in the system even after a reset. The code I write for EEPROM involves a lot of read/write functions, parsing of information, and creating "objects" in a non-OOP environment.
Languages: C
CLI: Makefiles
Tools: TTC Downloader, Realterm(for serial, but we use CAN), Sublime Text 3, PCAN-View, PCAN drivers, Tasking v3 Toolset
